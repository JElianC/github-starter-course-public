# Plan & Document vs. Agile

*Plan & Document*
- most of the time unsuccessful/overbudget
- prioritizes documentation

*Agile*
- customer is involved more
- prioritizes working software


| P&D | Agile |
| ----------- | ----------- |
| Plan | Change |
| P&D sticks to plans | Changes based on customers needs |
